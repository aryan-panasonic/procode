# Data Model Reference
## Entity Overview
ISA uses PostgreSQL as its primary data store with the pgvector extension for vector similarity search.
## Entity Relationship Summary
```
COUNTRY
  └── TENANT (many per country)
        └── STORE (many per tenant)
              └── PLANOGRAM (many per store)
                    └── SEGMENT
                          └── FIXTURE
                                └── POSITION → PRODUCT

TENANT
  └── TENANT_PRODUCT (junction: tenant ↔ product)
  └── PRODUCT_EMBEDDING (vector index per tenant's products)
  └── CAPTURE_ANALYSIS (stored results of shelf captures)

OAUTH_CLIENT (API credentials, not linked to tenant in schema)
```

## Entities
### country
Represents a geographic country used for tenant and store scoping.
| Field | Type | Notes |
|-------|------|-------|
| id | string | ISO or custom country code |
| name | string | Country name |
| iso | string | ISO code |
### tenant
Represents a retailer or isolated organization instance.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Unique tenant identifier |
| name | string | Retailer/organization name |
| country_id | FK | References `country` |
**Note**: `vendor_id` was removed in a past migration. Docs referencing `vendor` are outdated.
### store
A physical retail location under a tenant. One default store is auto-created on tenant creation.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Store identifier |
| tenant_id | FK | References `tenant` |
| country_id | FK | References `country` |
| name | string | Store name |
### product
A product in the master catalog. Shared across tenants via `tenant_product`.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Product identifier (barcode or custom) |
| name | string | Product name |
| barcode | string | Optional barcode |
### tenant_product
Junction table linking products to tenants.
| Field | Type | Notes |
|-------|------|-------|
| tenant_id | FK | References `tenant` |
| product_id | FK | References `product` |
### planogram
The expected shelf layout for a store.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Planogram identifier |
| store_id | FK | References `store` |
| name | string | Layout name |
### segment
A horizontal section (shelf level) within a planogram.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Segment identifier |
| planogram_id | FK | References `planogram` |
| order | int | Vertical ordering within planogram |
### fixture
A shelf unit within a segment.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Fixture identifier |
| segment_id | FK | References `segment` |
| fixture_type | int | Type identifier; type `11` receives special processing |
### position
A specific slot within a fixture where a product is expected.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Position identifier |
| fixture_id | FK | References `fixture` |
| segment_id | FK | References `segment` |
| product_id | FK | References `product` |
| order | int | Horizontal ordering within fixture |
### product_embedding
Stores image feature vectors for similarity search. One or more rows per product per tenant.
| Field | Type | Notes |
|-------|------|-------|
| id | UUID | Auto-generated |
| tenant_id | FK | Tenant scope |
| product_id | FK | References `product` |
| embedding | Vector(256) | Image feature vector (cosine distance search) |
| product_info_vector | Vector | OCR/text feature vector for reranking |
| metadata | JSON | Source image info, rotation, variant details |
**Note**: Vectors are stored directly in PostgreSQL via pgvector — ISA does NOT use a separate vector database (e.g. Qdrant). Any documentation referencing Qdrant is outdated.
### capture_analysis
Stores the result of a shelf analysis capture for later retrieval.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Application-generated capture ID |
| tenant_id | FK | References `tenant` |
| store_id | FK | References `store` |
| planogram_id | FK | References `planogram` |
| result_blob_url | string | Azure Blob URL for full JSON result |
| image_blob_url | string | Azure Blob URL for annotated shelf image |
| crops_blob_url | string | Azure Blob URL for per-product crop images |
| alternatives_blob_url | string | Azure Blob URL for alternative-product JSON |
| created_at | timestamp | Capture submission time |
### oauth_client
API client credentials for authentication.
| Field | Type | Notes |
|-------|------|-------|
| id | string | Client identifier (`clientId`) |
| secret_hash | string | Bcrypt-hashed client secret |
| scopes | JSON | Assigned scopes (stored but not enforced at route level) |

## Vector Storage Details
- **Extension**: pgvector installed via `init-db.sql`
- **Image vector**: `product_embedding.embedding` — `Vector(256)`, generated by OpenVINO embedding model
- **Text vector**: `product_embedding.product_info_vector` — generated by Azure OpenAI text embeddings from OCR output
- **Search method**: Cosine distance (`cosine_distance`) via SQLAlchemy/pgvector
- **Scope**: All vector searches are tenant-scoped — no cross-tenant leakage
## Blob Storage Layout (Azure Blob Storage)
| Container | Content |
|-----------|---------|
| Shelf images | Original uploaded shelf photos |
| Analysis results | JSON analysis output per capture |
| Annotated images | Shelf images with detection overlays |
| Crop images | Per-product crop sub-images |
| Alternative products | JSON of near-match alternatives per capture |
| Price results | OCR price extraction results |
| ML models | Embedding and detection model files (downloaded at service startup) |
## Migration History Notes
- `vendor_id` was part of `tenant` schema — removed in migration `2025-05-05`
- `product_embedding` table added in migration `2026-01-26`
- Database upgrades run as an `initContainer` in the `planogram` Kubernetes deployment, ensuring migrations apply before API startup
